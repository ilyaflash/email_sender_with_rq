Два конкурентных потока обеспечиваются с помощью запуска двух воркеров с помощью supervisord. Для этого в терминале требуется ввести команду "supervisord -n". В файле "supervisord.conf" можно настроить количество воркеров обрабатывающих очередь.
Для простоты тестирования, конфигурации писем берутся из файла json, функцией get_json_confs().
